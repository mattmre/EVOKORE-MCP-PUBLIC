Skill spotlights

Twelve skills worth knowing about

EVOKORE indexes 266 skills across 16 active categories. This page picks 12 representatives that show the range — code-heavy workflows, content workflows, planning skills, and meta-utilities. Every skill card includes one example invocation and a "best for" tag so you can scan to the ones that match the work you do.

Reading the cards

Code

Code workflow

Designed for repo work. Tends to touch git, run tests, produce PRs, and follow code conventions.

Non-code

Non-code workflow

Designed for content, planning, research, or operations. May produce markdown, PRDs, briefings, decision logs.

Both

Both

A meta-utility or planning skill that applies equally well to code and non-code work.

The twelve

Both

resolve_workflow

Category: Native tool (SkillManager)

One-sentence purpose: resolve a natural-language objective to the right skills, with reasoning attached.

Example invocation:

I need to wrap this session and leave a
clean handoff for tomorrow.

Best for: when you don't yet know which skill to invoke. Returns a ranked list with Why matched: explanations.

Code

repo-ingestor

Category: General Coding Workflows

One-sentence purpose: walk a repo end to end, summarize its shape, and produce a high-signal narrative so a downstream agent can act without re-reading every file.

Example invocation:

Ingest the current repo and produce a
shape summary I can hand to a refactor
panel.

Best for: the first turn of any agent that needs grounded repo understanding. Followed by a panel review on the artifact.

Code

improve-codebase-architecture

Category: Architecture

One-sentence purpose: surface architectural friction inside a bounded context and turn shallow modules into deep ones, anchored on the ADR for the affected context.

Example invocation:

Improve the session-isolation context.
The HTTP transport state is leaking
across connections.

Best for: module-scoped refactors. When 3+ viable interface shapes appear, escalates to the design-an-interface panel.

Code

triage-bug

Category: QA

One-sentence purpose: triage a bug from session evidence (replay log, evidence log, telemetry, repo audit) and produce a TDD fix plan plus a docs/bugs/<slug>.md artifact.

Example invocation:

Triage the rate-limiter regression from
last night's session. Produce a TDD plan
and the docs/bugs file.

Best for: the first 10 minutes after a bug report. Builds the failing-test seed before any fix attempt.

Code

tdd

Category: Orchestration Framework / commands

One-sentence purpose: enforce the Red → Green → Refactor cycle with explicit red-commit-hash evidence in the verification log.

Example invocation:

Apply tdd to the fix described in
docs/bugs/rate-limiter-regression.md.

Best for: any bug fix that touches behavior. Refuses to advance without the red reference.

Code

pr-manager

Category: General Coding Workflows

One-sentence purpose: package a working branch into a PR with the metadata, template sections, and reviewer hints the repo expects.

Example invocation:

Open a PR for the current branch using
the project's PR template. Include the
verification log link and the panel
review SHA.

Best for: the last 5 minutes of a coding session. Catches stale placeholders before reviewers do.

Non-code

to-prd

Category: Planning

One-sentence purpose: synthesize a PRD from prior session evidence and panel-review it before declaring it done.

Example invocation:

Turn the last two weeks of
session-replay JSONL into a PRD for the
HTTP-transport hardening initiative.

Best for: when "we should write this down" surfaces a few weeks into iteration. Evidence-driven, not aspirational.

Non-code

to-issues

Category: Planning

One-sentence purpose: decompose a PRD into AFK/HITL-tagged vertical-slice issues with acceptance criteria and panel-review notes attached.

Example invocation:

Decompose docs/prd/http-hardening.md
into issues. Use vertical slices; tag
AFK candidates.

Best for: turning a clean PRD into a roadmap an agent fleet can actually pick up.

Both

panel-of-experts

Category: Orchestration Framework

One-sentence purpose: assemble a virtual expert panel with defined personas, lenses, biases, and challenge prompts, then run the 10-step cycle to produce auditable findings.

Example invocation:

Invoke the Code Refinement panel on
src/SessionIsolation.ts. Full 10-step
cycle at depth 3.

Best for: when a single AI opinion is not enough — high-stakes architecture, security, planning, or stakeholder communication.

Both

aep-framework

Category: Orchestration Framework

One-sentence purpose: organize a remediation sweep into Align / Execute / Prove phases with severity-tiered closes and verification evidence per item.

Example invocation:

Run an AEP cycle on this refinement
report. Lock scope, then execute the
Critical tier first.

Best for: end-of-phase sweeps and audit-grade roadmap closes.

Non-code

content-research-writer

Category: Research and Content

One-sentence purpose: produce evidence-grounded long-form content by separating research, synthesis, and writing into distinct phases.

Example invocation:

Write a deep-dive on MCP transport
modes. Use the architecture docs in
this repo as primary sources.

Best for: long-form posts, internal explainers, and conference talk drafts. Pairs well with the Documentation panel as a final gate.

Non-code

meeting-insights-analyzer

Category: Research and Content

One-sentence purpose: extract decisions, owners, deadlines, and unresolved questions from meeting transcripts or research review notes.

Example invocation:

Analyze the persisted panel review at
docs/panel-reviews/2026-05-20/presentation-*.md
and extract decisions and follow-ups.

Best for: closing the loop after a panel run, a research review, or a stakeholder meeting.
