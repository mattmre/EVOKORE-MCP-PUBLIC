# Definition of Done (DoD)

Use this checklist for every task in `TASKS.md`.

## Required
- Acceptance criteria are explicitly listed and met.
- Diff is minimal and scoped to the task.
- Tests run (or rationale recorded if not possible).
- Results reported with commands and outcomes.
- Reviewer feedback captured (if required by risk triggers).
- TASKS updated with status, owner, and verification evidence.

## Optional (when applicable)
- DECISIONS updated for architectural changes.
- Documentation updated to match behavior changes.
- Follow-up tasks created for deferred scope.
