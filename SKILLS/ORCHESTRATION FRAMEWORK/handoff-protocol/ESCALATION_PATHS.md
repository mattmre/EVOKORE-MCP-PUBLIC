# Escalation Paths

Use this document to decide when and how to escalate issues.

## Escalate to Director
- Scope changes affecting multiple repos.
- Re-prioritization that displaces committed work.
- Conflicts between teams or owners.

## Escalate to Orchestrator
- Ambiguous acceptance criteria or blocking dependencies.
- Conflicting task ownership.
- Risk triggers that require review coordination.

## Escalate to Reviewer
- Security-sensitive changes.
- Schema or data model changes.
- Public API or interface changes.
- CI/CD or deployment pipeline changes.

## Review Escalation Criteria
- Risk triggers match review checklist matrix.
- Changes affect external dependencies or compliance.
- Work crosses multiple repos or teams.

## Escalate to Stakeholders
- Changes to external contracts or compliance requirements.
- Long delays or repeated blocked status.
